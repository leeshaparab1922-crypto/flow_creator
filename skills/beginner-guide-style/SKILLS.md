---
name: beginner-guide-style
description: Style rules for producing a beginner-friendly getting-started guide — define every term on first use, annotate every code block, and write outlines as binding contracts with no prose. Use when outlining, drafting, reviewing, or editing a guide aimed at readers with zero prior knowledge of the topic.
---

Apply these rules to any outline, draft, review, or edit of a beginner-facing guide:

- **Define every term on first use.** Never assume prior knowledge of the topic. The first time a technical term, tool, or concept appears, explain what it is in plain language before using it again.
- **Annotate every code block.** Every code example must be complete and runnable, with an explanation of what each meaningful line or block does. Never paste a code sample without walking through it.
- **No prose in outlines.** An outline is a binding contract, not a draft. Each section entry must specify the section title, its learning objective, the concepts it covers, any code examples to include (described, not written out), and a target word count. Do not write narrative or explanatory sentences inside an outline.
